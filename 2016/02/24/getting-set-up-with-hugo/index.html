Getting Set Up With Hugo

2016-02-24 · 198 words · 1 minute read
Web
Hugo

Motivation

There are certain instances where statically generating your website is a great idea. There are tools out there like WordPress, Drupal, and Joomla that provide much functionality that can be overkill. The difference between Hugo and these Content Management Systems (CMS) is static vs. dynamic content delivery. While it does have its restrictions, static site delivery has some pretty great performance benefits. If the server doesn’t have to query a database and build the page, it will deliver the user content much quicker. The idea is that we do all the work in building the site once and deliver that product many times.

Installation

Installation steps are officially documented here.

Quickstart guide is located here.

Cheat Sheet

Objective Command
Create new site hugo new site path/to/site
Create post hugo new post/example.md
Run server hugo server
Run server with theme hugo server --theme=<theme name>
Run server and watch files for updates hugo server -w

Notes

Get All Themes

Steve Francia, the founder lof the Hugo project, maintains a repository of all hugo themes here. You can conveniently download all the themes and add them to your poject.

$ cd /path/to/project
$ git clone --depth 1 --recursive https://github.com/spf13/hugoThemes.git themes
